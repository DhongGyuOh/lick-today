# AlphaTex 작성 규칙 (Lick Today 프로젝트용)

이 프로젝트의 `data/licks/*.json`에 들어가는 `alphaTex` 필드를 작성할 때 지키는 규칙입니다.
Dify 프롬프트에 그대로 포함시켜서 AI가 처음부터 유효한 문법으로 생성하도록 유도하는 용도입니다.

## 1. 기본 구조

```
\title "제목"
\tempo 120
.
마디1 내용 |
마디2 내용 |
```

- `\title`, `\tempo` 등 메타데이터 → `.` (점 하나, 구분자) → 마디(bar) 내용 순서
- 마디는 `|`로 구분. **마지막 마디 뒤에는 `|`를 붙이지 않는다.**
- 한 줄에 마디를 여러 개 이어써도 되고 줄바꿈으로 나눠도 됨 (줄바꿈은 `\n`으로 표현)

## 2. 음표(노트) 표기

기본형: `프렛.줄.박자`

```
0.6.4   → 6번줄(로우E) 개방, 4분음표
3.6.8   → 6번줄 3프렛, 8분음표
```

- 줄 번호는 **6번줄이 로우 E(가장 두꺼운 줄), 1번줄이 하이 E(가장 얇은 줄)** — 실제 기타 연주 시 육안으로 보는 순서와 반대이니 주의.
- 쉼표(rest)는 `r` (박자 필요하면 `r.8` 처럼 붙임)

### `:duration` 접두사로 반복 지정 (권장)

같은 박자가 여러 개 이어질 땐 매번 `.박자`를 안 써도 됨:

```
:8 3.3 5.3 2.2 4.2 |
```

→ 이 마디의 모든 노트가 8분음표. **이펙트를 쓸 노트가 있으면 이 방식을 쓰는 게 훨씬 안전함** (아래 3번 참고).

## 3. 노트 이펙트 — 여기서 실수가 가장 많이 났음 ⚠️

이펙트는 `{}`로 표기하며 **노트 뒤에 바로 붙인다.**

| 이펙트 | 표기 | 의미 | 주의사항 |
|---|---|---|---|
| 해머링/풀링 | `{h}` | 피킹 없이 다음 음으로 이어짐 | 해머온·풀오프 **둘 다 `{h}` 하나로 통일**. `{p}`는 없음 (다이내믹 마킹으로 잘못 해석됨) |
| 슬라이드 | `{sl}` | 다음 음으로 미끄러짐 | **출발(시작) 음에 붙인다.** 도착 음에 붙이면 안 됨. 예: `3.3{sl} 5.3` (3프렛→5프렛 슬라이드) |
| 벤드 | `{b (시작 끝)}` | 피치 벤드 | 단위는 쿼터톤(4분의1음). `(0 4)` = 반음 2개(온음) 벤드, `(0 2)` = 반음 1개 벤드 |
| 비브라토 | `{v}` | 비브라토 | 단독으로 사용 |
| 팜뮤트 | `{pm}` | 팜뮤트 | |

**절대 규칙: `프렛.줄.박자` 3파트 형식에는 `{}`를 절대 붙이지 않는다.**
`{}`가 붙는 순간 duration 파트가 없어야 정상 해석됨. 그래서 이펙트를 쓸 마디는 항상
`:duration` 접두사(위 2번) + `프렛.줄{이펙트}` 형태로 쓴다.

```
❌ 5.3.8{sl}         → "Unrecognized property 'sl'" 에러
✅ :8 5.3{sl} 3.3    → 정상
```

여러 이펙트를 동시에 걸 수도 있음: `8.3{b (0 4)}{v}` (벤드 + 비브라토)

## 4. 더블스탑 / 코드 (동시에 여러 음)

괄호로 묶고, 박자는 괄호 뒤에 붙임:

```
(0.2 0.1).4        → 2번줄 개방 + 1번줄 개방, 동시에, 4분음표
(2.2 0.1){v}        → 위와 동일하되 :duration 접두사 방식 + 비브라토
```

## 5. 체크리스트 (AI 프롬프트에 그대로 넣기 좋은 요약)

- 마디는 `|`로 구분, 마지막 마디 뒤엔 `|` 없음
- 줄 번호: 6=로우E ~ 1=하이E
- 이펙트 쓸 마디는 `:duration` 접두사 + `프렛.줄{이펙트}` 형식만 사용 (3파트 `프렛.줄.박자{이펙트}` 금지)
- 해머온/풀오프는 항상 `{h}` (별도 `{p}` 없음)
- 슬라이드 `{sl}`은 출발 음에 표기
- 벤드는 쿼터톤 단위: `{b (0 4)}` = 온음 벤드

## 6. 커밋 전 검증 (필수)

`scripts/validate-alphatex.mjs`로 실제 alphaTab 파서에 태워서 문법 오류를 미리 잡을 것:

```bash
node scripts/validate-alphatex.mjs data/licks/2026-07-05.json
```

모든 릭이 `OK`로 나와야 커밋. `FAIL`이 뜨면 에러 메시지의 라인/컬럼 위치를 보고 위 표를 참고해 수정.
드럼 트랙이 있으면 `OK` 라인 끝에 트랙 이름이 같이 출력됨 (예: `트랙: Guitar, Drums(드럼)`).

## 7. 드럼 백킹 트랙 추가하기 (멀티트랙)

한 `alphaTex` 문자열 안에 기타 트랙과 드럼 트랙을 함께 작성할 수 있습니다. `\track` 태그로 트랙을 나누고,
드럼 트랙은 `\instrument percussion`을 지정합니다.

```
\title "제목"
\tempo 120
.
\track "Guitar"
:8 0.6 3.6 0.5 2.5 0.6 3.6 0.5 2.5 |
\track "Drums" \instrument percussion \clef neutral
:8 36 42 38 42 36 42 38 42 |
```

- 첫 번째 `\track "Guitar"`는 생략해도 되지만, 명시하면 UI에 트랙 이름이 표시되므로 **항상 써주는 걸 권장**.
- **드럼 노트는 프렛/줄이 아니라 GM(General MIDI) 퍼커션 번호**를 그대로 씀. 이름(예: `KickHit`)이 아니라 **숫자**를 쓰는 게 훨씬 안전함 (버전에 따라 이름 매핑이 다를 수 있음).
- 마디 개수는 기타 트랙과 드럼 트랙이 **같아야** 함 (`|` 개수를 맞출 것).

### GM 퍼커션 번호 참고표 (자주 쓰는 것만)

| 번호 | 악기 |
|---|---|
| 35, 36 | 베이스 드럼(킥) |
| 38, 40 | 스네어 |
| 42 | 하이햇 클로즈 |
| 44 | 하이햇 페달 |
| 46 | 하이햇 오픈 |
| 49 | 크래시 심벌 1 |
| 51 | 라이드 심벌 1 |
| 41, 43, 45, 47, 48, 50 | 톰(로우~하이) |

가장 기본적인 블루스 셔플 드럼 패턴 예시(8비트, 킥-하이햇-스네어-하이햇 반복):

```
\track "Drums" \instrument percussion \clef neutral
:8 36 42 38 42 36 42 38 42 |
```

### JSON에 표시하기

드럼 트랙이 포함된 릭은 `hasDrums: true`를 추가해주세요 (없으면 `false`로 취급됨, 하위호환 유지):

```json
{
  "id": "...",
  ...,
  "hasDrums": true,
  "alphaTex": "..."
}
```

이 값은 UI에서 배지 표시나 필터링에 쓰입니다. **실제 재생 시 트랙 목록(믹서 패널: Mute/Solo/Volume)은
`alphaTex`를 파싱한 결과에서 자동으로 만들어지므로, `hasDrums`는 오직 메타데이터 용도**입니다 —
값을 빼먹어도 재생 자체는 정상 동작합니다.

## 8. 배포 관련

이 프로젝트는 GitHub + Vercel로 배포됩니다. `main` 브랜치에 push하면 Vercel이 자동으로 빌드/배포합니다.
별도의 GitHub Pages용 `build:gh` 스크립트나 basePath 설정은 더 이상 필요 없습니다.
